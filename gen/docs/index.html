Receipt Printer API Documentation

GET /login

Starts OAuth authorization code flow. After this is complete, a client application will be allowed to make authenticated HTTP requests to Receipt API Server by calling `fetch()` with the option `{credentials: 'include'}`.

Parameters:

  • redirect_uri: URL? - The URL to redirect back to after authentication is complete. Must be a *.recurse.com subdomain.

POST /text

Content-Type: application/json OR application/x-www-form-urlencoded

Print text to the printer.

Parameters:

  • text: string - May only contain ASCII characters in the range 32-126.
  • bold: boolean? -

POST /image

Content-Type: image/jpeg OR image/png OR image/gif OR application/octet-stream

Print an image. Only accepts P4 .pbm format right now. (Future: If it's wider than 512px across, it will be shrunk down to be 512px wide, preserving aspect ratio. If it's not black and white, it will be converted to a black and white dithered image.)

POST /escpos

Content-Type: application/octet-stream

Send raw ESC/POS bytes to the printer. Pass the ESC/POS bytes directly in the request body.
