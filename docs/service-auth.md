# Introduction

We use HTTP Message Signatures to authenticate services to the Receipt Printer API.

If you would like to add a new service, you can do the following:
- generate a new key pair (using, e.g., `node ./scripts/generate-key-pair.js outfname`)
- create a pull request adding your new public key to the repo
- add the private key to your server

When your service sends requests to the Receipt Printer API server, it must
include a "Signature" header generated by signing the request body with the
private key and then getting the signature's base64 representation.

When the server receives new requests, it tries to verify the signature against
its known public keys; only requests whose signatures can be verified by a
known key will be accepted.
